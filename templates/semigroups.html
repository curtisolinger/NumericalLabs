Numerical Semigroup Invariants and the Second Frobenius Number

{% endblock %} {% block content %}

Imagine in the country you live, the currency is only made up of 3 and 5 cent coins. Ignoring the question of why your country has only these two denominations, you might run into the situation of trying to get change for an 8 cent candy bar if you only have two 5 cent coins. You might give the store clerk 10 cents, but could you expect to get back your 2 cents of change? In this country, no you couldn't.

In a 3 and 5 coin system, there are values of money that cannot be created such as 2, 3, 4, and 7 cents. But every value after that is obtainable if you have enough 3 and 5 cent coins.

Trying to determine the largest monetary value not obtainable for a country's coin system is an example of the Frobenius Coin Problem. The answer to that question is called the Frobenius Number, named after Ferdinand Georg Frobenius. He was a German mathematician born February 14, 1877, in a suburb of Berlin. Frobenius worked on diverse fields such as elliptic functions, differential equations, number theory, and group theory and taught at the University of Berlin and the ETH Zurich.

If there are only two denominations of coins \(x\) and \(y\), the Frobenius Number is given by the equation \(xy - x - y \). In our 3 and 5 cent system, the Frobenius number is \(3 \times 5 - 3 - 5 = 7\).

For a three or more coin system, there is no explicit answer nor are there any fast and efficient computational algorithms for computing the answer.

Generators

The coin denominations in a country's currency system are examples of generators as they can ``generate'' all possible values of money if you have enough coins.

If your generators are 3 and 5, repeated addition of these numbers will result in the following first ten terms:

3, 5, 6, 8, 9, 10, 11, 12, 13, 14.

Every number except for 1, 2, 4, and 7 is in this list.

Using mathematical notation we would write this set as \[ \langle 3, 5 \rangle = \{3, 5, 6, 8, 9, 10, \ldots \} \] where \( \langle 3, 5 \rangle\) denotes the set generated by 3 and 5.

The numbers we cannot obtain are written as \[ \mathbb{Z^+} \setminus \langle 3, 5 \rangle = \{ 1, 2, 4, 7 \} \] where \(\mathbb{Z}^+\) is the set of positive integers. This set is finite which is an important observation we will come back to later.

It is not always the case that the complement is finite. If we take 2 as a generator, we get all the positive even numbers. Its complement in the set of positive integers is not finite.

To get a feeling for the numbers generated by a set of generators, try entering in your own generators and see which numbers they produce.



Numerical Semigroups

By examining all the values we can create with whatever denominations of coins we have in our currency system, as long as the coins themselves don`t share any common factors, we have been creating a structure called a numerical semigroup.

Formally, a numerical semigroup is a subset of the nonnegative integers with addition as the binary operation and a finite complement within the positive integers.

Thus 3 and 5 create a numerical semigroup but 2 does not. In fact, all you need for a set of generators to create a numerical semigroup is for the generators to be relative prime, that is, they only share a common factor of 1.

Next notice that every numerical semigroup has a set of generators. The easiest way to see this is to consider the smallest non-zero element in the semigroup. This element, together with all other elements in the semigroup that are not divisible by it, form a set of generators of the semigroup.

This is because every element in the semigroup can be expressed as a linear combination of the generators with non-negative integer coefficients. Thus, every numerical semigroup does indeed have a set of generators.

While every numerical semigroup contains a set of generators, the inverse that every set of generators creates a numerical semigroup is not always true. The even integers generated by 2 is an example of this.

Numerical Semigroups are the main field of study for this article and while they might seem simple, there is a great deal to discover about them. To get started, we will lay out some definitions that will allow us to examine numbers, called invariants, that capture information about the structure of the semigroups.

Max and min factorization lengths

Let \( m_1, \ldots, m_n \) be a set of generators for the numerical monoid \( M \). Then, for any monoid element \(m \), the set of factorizations of \(m \in M \) is \[ \mathsf{Z}(m) = \left \{ (a_1, \ldots, a_k) \mid m = \sum_{i=1}^{k} a_im_i \right \} \] and the set of factorization lengths of \(m \in M\) to be \[ \mathsf{L}(m) = \left\{\sum_{i = 1}^{k} a_i : (a_1, \ldots, a_k) \in \mathsf{Z}(m)\right\}. \]

For example, in the monoid generated by 2 and 3, the element 6 has the following two factorizations (3, 0) and (0, 2) since \[ (3, 0) \cdot (2, 3) = 3 \times 2 + 0 \times 3 = 6 \] and \[ (0, 2) \cdot (2, 3) = 0 \times 2 + 2 \times 3 = 6 \] where \( \cdot \) is the dot product operation from linear algebra. Thus \[ \mathsf{Z}(6) = \left \{ (3, 0), (0, 2) \right \}. \] The element 6 then has the set of factorization lengths of \[ \mathsf{L}(m) = \{ 3, 2 \} \] and it has a maximum factorization length 3 and a minimum factorization length of 2.

An interesting question regarding these max factorization lengths is for any max factorization length k, how many monoid elements have that max factorization length?

For example in the monoid generated by 2 and 3, 6 has a max factorization length of 3 but so does 5. By looking at the max factorization lengths for the nearby elements, it is the case that there are only two elements that have a max factorization length of 2.

The Second Frobenius Number

Another situation that arises is how to use up all the change one collects usually in a kitchen catch-all drawer. If I know the price of something ahead of time, I might try to arrive at that value using the greatest number of coins possible. Thus, reducing the total number of coin I have at home.

If something cost 15 cents and I have many 3 and 5 cents coins, I could take to the store five 3 cent coins instead of only three 5 cent coins, which would get me closer to my goal of a coin free existence. In other words, we want to bring to the store the coins that have a max factorization length for a given value.

In this example, 15 has a max factorization of five. Are there any other values that have that same max factorization length? Yes, 17 also have a max factorization length of five as \[ 4 \times 3 + 1 \times 5 = 17. \]

Also, 19 has a max factorization length of five \(3 \times 3 + 2 \times 5 = 19 \). But 15, 17, and 19 are the only values that have a max factorization length of five. Furthermore, the number of elements that have a max factorization length of six is also three, as is the number of elements with a max factorization length of seven and so on. You would say the number of elements with a max factorization length of \(k\) stabilizes. In this numerical semigroup with 3 and 5 as generators, that number is three.

This raises an interesting question: given an arbitrary coin system with any number of denomination of coins, does the number of values of money you can create with a max factorization length of k always stabilize at some value. The answer is yes, and the number at which it stabilizes is called The Second Frobenius Number.
